# 全局编码标准 (Global Engineering Standards)

所有源代码文件均须遵守以下基本准则。

## 1. 命名规范 (Naming)
- **变量与函数**: 使用小驼峰式 (`lowerCamelCase`) 或蛇形命名 (`snake_case`)，视具体语言惯例而定。
- **类与类型**: 使用大驼峰式 (`PascalCase`)。
- **常量**: 使用大写蛇形命名 (`UPPER_SNAKE_CASE`)。

## 2. 代码组织 (Organization)
- **单一职责**: 每个函数不应超过 30 行，每个类不应超过 300 行。超过则必须进行重构或拆分。
- **无硬编码**: 所有的 Magic Numbers (魔法数)、API 端点、配置项必须提取到常量文件或环境变量中。

## 3. 注释与文档 (Documentation)
- **为什么，而不是是什么**: 注释应解释代码背后的决策逻辑，而非描述代码的行为。
- **API 暴露**: 所有导出的函数、类、接口必须包含 JSDoc / Docstring / 注释头，描述参数和返回值。

## 4. 安全 (Security)
- **严禁提交机密信息**: 绝不允许在代码中包含 API Key、密钥、密码或敏感环境变量。
- **输入校验**: 任何来自外部的输入（API 请求、用户输入）必须在处理前进行验证。
