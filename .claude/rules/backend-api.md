# 后端 API 与 核心逻辑标准 (Backend & Core Rules)

适用于 `src/api/**` 和 `src/core/**` 目录。

## 1. 错误处理 (Error Handling)
- **非沉默失败**: 严禁捕获异常后不作处理或直接忽略。
- **结构化响应**: 所有接口必须返回统一的 JSON 格式。成功时返回数据，失败时返回错误代码和描述。
- **日志记录**: 每个错误必须记录在日志中，包含错误堆栈和上下文信息。

## 2. 数据库访问 (Database Access)
- **避免 N+1**: 在处理批量数据时，必须进行关联查询 (Joins / Includes)，严禁在循环内执行查询。
- **事务一致性**: 涉及多个表的更新必须包含在原子事务中。
- **参数化查询**: 严禁直接拼接 SQL 字符串，必须使用 ORM 或参数化查询以防止 SQL 注入。

## 3. API 契约一致性 (Contract Alignment)
- **ADR 优先**: 在编写 API 端点前，必须验证其是否符合 `docs/arch/` 中的定义。
- **参数验证**: 使用 `pydantic`, `zod`, `joi` 等验证库对所有进入 API 的数据进行强类型校验。

## 4. 幂等性 (Idempotency)
- **POST/PUT**: 对于非查询操作，应考虑设计幂等性机制（如使用 UUID/Token 防止重复请求导致数据重复）。
